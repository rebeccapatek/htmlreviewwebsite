HTML

Provides the structure to the content appearing on a website

HTML stands for HyperText Markup Language

Now what is a markup language you may be asking yourself? A markup language is a computer language that defines the structure and presentation of raw text. HTML allows a computer to interpret raw text that is wrapped in HTML elements Hypertext is text displayed on a computer or device that provides access to other text through links (also known as hyper links)

Let's Talk about elements

Html is composed of elements. These elements structure webpage and define its content.
EX: a paragraph element. A paragraph element is made up of
  1. Opening tag (<p>)
  2. Content (the text that you are displaying on the webpage. Ex: "Hello World!"
  3. Closing tag (<p>)

So basically the HTML element is a unit of content in an HTML document formed by the HTML tags and the text or media that it contains.

The body element is a key HTML element used to build a webpage. Only content inside the opening and closing body tags can be displayed on the screen. (although some browsers do accommodate for code outside the body element. So usually most of the code on your screen is nested in the body element.

Let's talk about Structure

When an element is contained inside of another element, it is considered the child of that element. So for example if you have a paragraph tag with content inside of a body element, then you would say that the paragraph is considered the "child" of the "parent" body element. You could also say that it is nested in the parent element.

For readability, it is common practice to add two spaces of indentation for the child of a parent element.

There can be multiple levels of nesting (grandchildren, great-grandchildren, etc.) The relationship between elements and their ancestors is known as hierarchy.

One of the reasons that understanding the HTML hierarchy is important, is because the child elements can inherit behavior and styling from their parent element.

Here's an example:

   <body>
     <h1></h1>
     <p></p>
   </body>

Both the header and the paragraph would be considered children of the body, while the h1 and p would also be considered siblings.

Let's talk about the div tag

A div is another important element in HTML. Div stands for division and they are defined as a container that divides the page into Section. These are very useful for grouping elements in your HTML together. Divs don't inherently have a visual representation, but when we want to apply custom styles using CSS to our elements, divs allow us to group the HTML elements and apply the same styles for all HTML elements inside.

Attributes are a way to expand an element's tag.

  • These are content added to the opening tag of an element.
  • These can be used to provide information or to change styling.

Attributes have two parts

  1. The name of the attribute. (ex: id)
  2. The value of the attribute. (ex: "intro")

So here's what a div would look like with this attribute:

   <div> id="intro">
     <h1>Introduction</h1>
   </div>
